## 1. 第一版范围

只做一个核心模块：

> 在线作品管理与展示

包含：

| 端     | 功能                               |
| ------ | ---------------------------------- |
| 前台   | 查看在线作品列表、点击访问         |
| 后端   | 提供作品查询接口                   |
| 数据库 | 存储作品信息                       |
| 管理端 | 第一版可先不做页面，用接口维护数据 |

## 2. 核心功能

### 2.1 前台作品展示

展示字段：

| 字段     | 说明                       |
| -------- | -------------------------- |
| 作品名称 | 项目名称                   |
| 作品简介 | 一句话介绍                 |
| 封面图   | 项目截图/封面              |
| 技术栈   | Vue、React、Spring Boot 等 |
| 标签     | 工具、AI、全栈、后端       |
| 在线地址 | 点击访问                   |
| 源码地址 | 可选                       |
| 部署类型 | 当前服务器 / 外部地址      |
| 状态     | 运行中 / 开发中 / 已下线   |

## 3. 部署地址设计

作品支持两种访问方式：

| 类型     | 说明           | 示例                         |
| -------- | -------------- | ---------------------------- |
| LOCAL    | 当前服务器部署 | `/apps/chat`                 |
| EXTERNAL | 外部地址       | `https://xxx.github.io/demo` |

前端点击时统一使用：

```
projectUrl
```

也就是不管作品部署在哪里，前端只关心一个访问地址。

## 4. 数据库表设计

### portfolio_project：在线作品表

```
CREATE TABLE portfolio_project
(
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',

    name VARCHAR(100) NOT NULL COMMENT '项目名称',

    description TEXT COMMENT '项目简介',

    cover_url VARCHAR(500) COMMENT '项目封面图片URL',

    tech_stack VARCHAR(500) COMMENT '技术栈，多个使用逗号分隔',

    tags VARCHAR(500) COMMENT '项目标签，多个使用逗号分隔',

    deploy_type VARCHAR(30) NOT NULL DEFAULT 'EXTERNAL'
        COMMENT '部署类型：LOCAL-当前服务器；EXTERNAL-外部地址',

    project_url VARCHAR(500) NOT NULL
        COMMENT '项目访问地址',

    source_url VARCHAR(500)
        COMMENT '项目源码地址',

    status VARCHAR(30) NOT NULL DEFAULT 'ONLINE'
        COMMENT '项目状态：ONLINE-运行中；DEVELOPING-开发中；OFFLINE-已下线',

    is_recommend TINYINT(1) NOT NULL DEFAULT 0
        COMMENT '是否推荐项目：0-否；1-是',

    sort_order INT NOT NULL DEFAULT 0
        COMMENT '排序值，越大越靠前',

    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
        COMMENT '创建时间',

    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
        ON UPDATE CURRENT_TIMESTAMP
        COMMENT '更新时间'

) COMMENT = '个人作品集-在线作品表';
```

## 5. 字段枚举建议

### deploy_type

| 值       | 说明       |
| -------- | ---------- |
| LOCAL    | 当前服务器 |
| EXTERNAL | 外部地址   |

### status

| 值         | 说明   |
| ---------- | ------ |
| ONLINE     | 运行中 |
| DEVELOPING | 开发中 |
| OFFLINE    | 已下线 |

## 6. 后端接口设计

### 6.1 前台查询作品列表

```
GET /api/portfolio/projects
```

返回：

```
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "id": 1,
      "name": "AI 聊天助手",
      "description": "基于大模型的在线聊天工具",
      "coverUrl": "/uploads/covers/chat.png",
      "techStack": ["Vue3", "Spring Boot", "PostgreSQL"],
      "tags": ["AI", "工具"],
      "deployType": "EXTERNAL",
      "projectUrl": "https://xxx.github.io/chat",
      "sourceUrl": "https://github.com/xxx/chat",
      "status": "ONLINE",
      "recommend": true
    }
  ]
}
```

### 6.2 前台查询作品详情

```
GET /api/portfolio/projects/{id}
```

### 6.3 管理端新增作品

```
POST /api/admin/portfolio/projects
```

### 6.4 管理端修改作品

```
PUT /api/admin/portfolio/projects/{id}
```

### 6.5 管理端删除作品

```
DELETE /api/admin/portfolio/projects/{id}
```

## 8. 推荐技术栈

结合你的情况，建议：

```
前端：Vue3 + Vite + TypeScript
后端：Spring Boot 3
ORM：MyBatis-Plus
数据库：MySQL
部署：Docker Compose + Nginx
```

## 9. 第一版不要做太多

第一版建议只做：

✅ 作品列表查询
 ✅ 作品详情查询
 ✅ 管理接口 CRUD
 ✅ 数据库存储
 ✅ Nginx 支持当前服务器项目代理
 ✅ 外部 URL 跳转

暂时不做：

❌ 用户注册
 ❌ 评论
 ❌ 点赞
 ❌ 博客
 ❌ 复杂权限
 ❌ 文件管理系统

## 10. 一句话新版需求

建设一个带后端和数据库的个人在线作品集网站，第一版只展示在线作品。作品信息存储在数据库中，由后端提供查询接口。每个作品支持配置在线访问地址，既可以指向当前服务器通过 Nginx 代理部署的应用，也可以指向 GitHub Pages、Vercel 或其他外部 URL。
