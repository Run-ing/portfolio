## Why

当前项目已经包含前端、后端和作品代理能力，但缺少一套明确的本地部署约定，导致在本机或服务器上部署时需要临时判断构建顺序、环境变量、端口和反向代理配置。该变更用于沉淀可重复执行的本地部署能力，让项目可以在本地环境中稳定构建、启动、验证和排障。

## What Changes

- 新增本地部署说明与脚本约定，覆盖依赖准备、环境变量配置、构建、启动、停止和健康检查。
- 明确前端静态资源、后端 API 服务、数据库连接和 Nginx 反向代理之间的部署关系。
- 提供本地部署配置模板，避免敏感信息进入仓库，同时让必填配置可被检查。
- 增加部署验证步骤，确认前端页面、后端接口、作品列表接口和代理路径可用。
- 不引入破坏性变更；现有开发启动方式和作品管理能力保持兼容。

## Capabilities

### New Capabilities

- `local-deployment`: 定义项目在本地或单机服务器上的构建、配置、启动、反向代理和验证行为。

### Modified Capabilities

无。

## Impact

- 影响部署文档、环境变量示例、启动脚本、Nginx 配置示例和部署验证脚本。
- 可能涉及前端构建产物路径、后端运行端口、数据库连接配置、API 基础路径和 `/apps/` 作品代理路径。
- 不改变现有业务 API 合约，不修改已存在的 `portfolio-projects` 需求。
